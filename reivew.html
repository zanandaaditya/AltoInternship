Section 2 Version Control and Branch Management

  • Saya tahu cara membuat branch profesional ketika kita bekerja dalam kerja tim.
  • Membuat branch tidak merusak main branch ketika kita menambahkan fitur atau apa pun, sehinnga kita membuat new branch dan kita dapat membuat apa pun di new branch.
  • Untuk membuat pekerjaan dengan baik, kita harus membuat main branch jika branch develop sudah jelas, branch develop jika kita menambahkan fitur baru atau apa pun sebelumnya ke main branch, dan menambahkan fitur 1,2,3, dll untuk menambahkan fitur, bagian, atau apa pun.

Section 3 Fundamental UI/UX

  • Dalam pekerjaan UI/UX akan ada pekerjaan yang lebih lingkup seperti UX Writer, UX Engginer, UI Designer, UX Researcher, Intercation Desginer. Namun masih ada beberapa perusahaan menggabungkan pekerjaan UI/UX Designer.
  • Dalam UI/UX terdapat basic role, yaitu memahami user, kejelasan sistem, menyerupai dunia nyata, kebebasan pengguna, konsisten, mencegah error, mengenali daripada mengingat, flexibel dan efisien, dan estetik ,serta minimalis.
  • Terdapat 3 user testing, yaitu offline dimana secara tatap muka seorang UI/UX Designer menemui seseorang yang akan mencoba aplikasi yang dibuat, online yaitu secara daring yang efesien dilakukan pada masa sekarang dan menghemat biaya, Geurilla yaitu siapapun bisa menjadi user testing dan aplikasi yang akan dicoba akan diletakkan di tempat umum dan biasanya akan mendapatkan reward setelah mengisi user testing.
